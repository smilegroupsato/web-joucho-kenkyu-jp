# writing-rules

本文整形のルール。

- 観測記録は軽いHTMLで書く。
- 段落は `p` タグに分ける。
- 観測終了は本文に入れる。
- コードを本文として出すときはコード表示にし、HTMLはエスケープする。
- 説明しすぎず、余白を残す。
- 観測記録ページには、判明している範囲で観測日時を表示する。
- 推測で日時を補わない。

## 観測日時

観測記録のメタ情報には、可能な限り観測日時を入れる。

表記例：

```text
observed: 2026-07-06 21:34 JST
```

Notion側に日付だけがある場合は、日付のみを表示する。

```text
observed: 2026-07-06
```

不明な場合は本文に入れず、`docs/remaining-tasks.md` に未補完として残す。

## Mobile

小幅画面で以下を確認する。

- 長いタイトルが画面外にはみ出さない。
- `pre.ascii-logo` は横スクロールまたは縮小で表示する。
- `.shelf-list` と `.record-list` は1列に崩れる。
- `.observation-meta` は折り返して読める。
